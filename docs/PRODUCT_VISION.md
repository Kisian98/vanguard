# Product Vision

Vanguard is a fictional field workstation platform for MilSim, role-play, demonstrations, and technical experimentation.

The project should be treated as both a technical build and a product-design portfolio piece.

---

## Vanguard Systems

In-universe, Vanguard OS is developed by **Vanguard Systems**.

Vanguard Systems is currently imagined as a private contractor focused on dependable field computing products. The company may eventually have multiple products, but only Vanguard OS is defined for now.

The company should feel competent, restrained, and engineering-led. Avoid making it feel like an exaggerated villain corporation.

---

## Vanguard OS

Vanguard OS is the current product focus.

It is a fictional operating environment built on NixOS and Hyprland. Its purpose is to provide a believable field workstation experience while remaining technically reproducible and usable.

The system is not intended to be a daily-driver desktop. Normal usability matters, but MilSim immersion and visual cohesion are the main priorities.

---

## Audience

The real-world audience is:

- the project owner,
- MilSim and role-play participants,
- technical observers,
- portfolio reviewers,
- and future collaborators.

The in-universe customer direction is:

- military-industrial organizations,
- private security companies,
- and field-oriented organizations that need rugged computing tools.

---

## Product Experience

A user should feel that Vanguard is:

- responsive,
- dependable,
- meaningful,
- actionable,
- and built for operators.

The ideal reaction is not only that Vanguard looks good, but that it feels useful and believable once interacted with.

---

## Implementation Philosophy

Vanguard should integrate existing software before replacing it.

Existing tools can be included when they fit the purpose. Custom components should be created when existing tools cannot deliver the required functionality, integration, or experience.

Vanguard should be offline-capable first. Internet access should enhance the system with public dashboards, OSINT tools, maps, and live feeds.

---

## Version 1.0

Vanguard 1.0 means the system is ready for its first event.

It does not mean the project is finished. It means the workstation is coherent, stable, and presentable enough for public use.

Demo builds and early previews should remain below 1.0.
